PETER LUK

Green Island

User Experience Design, User Interface Design

Green Island is an application with a goal of changing the backpacking experience, allowing the user to have a better access to information as well as creating a new game like mechanic for the young millennial travellers that is engaging and fun while also able to help them save money at the same time. To understand the game mechanic behind backpacking, we have to understand why the user base enjoys from the activity. I took on the project of creating guidelines in the area of Branding, UI components, and Layout.















This project came off as a new challenge for me, essentially, the client want the application to be completed in a matter of days as they wish to begin production of the app as soon as possible. Initial research was done through looking branding/application guidelines that are published for other reputable organizations. The breadth and depth of each guideline varies, so identifying the right topics to cover in Green Island’s guideline was crucial. Due to the constraints in time, I had to forego defining a structural skeleton using Axure and went straight into mid fidelity wireframing with Illustrator, iterating the look and UI components of the layout. As the client wishes to proceed with the application, my main duty was to help create a UI and layout guideline which the developers and marketing team can use.







When designing the user flow, our primary goal is to start understanding the objective of what the user is trying to achieve. In the case of Green Island, we had to understand what certain utility tools that travellers would use often are and the desire or needs the user might want to satisfy. For example, while incorporating the game mechanics of routes and badges, it’s important to understand how backpackers decide on the route they choose and what type of information are driving them to certain actions. Casual users may want to browse through the prices while experts would definitely want a challenge, so in order to be able solve specific problems, it’s important to create options of shorting information into categories.

By allowing the users the ability to control how many tabs they want open as well as easy access to certain tabs through a click of a button would give them a smooth transition from jumping from screen to screen and tabs to tabs. However, this still doesn’t solve one of the biggest issues that the current interface is facing which is the lack of spaces and access to particular files. In order to allow the user a better flow and experience, it’s important to have multiple views of the interface.
